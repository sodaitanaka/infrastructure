### 【4】スプレッドシートの編集範囲制限

#### 実験で判明した事実

| 事項 | 内容 |
|------|------|
| Drive共有だけでは不可 | ファイル単位の権限（reader/writer）のみ。セル単位の制御はできない |
| テンプレートの保護設定はコピーに引き継がれない | コピー後に毎回APIで設定し直す必要がある |
| `addProtectedRange` だけでは不十分 | `editors.users` を明示しないと、writerユーザーが保護範囲も編集できてしまう |
| 既存保護は削除してから再作成 | 再実行時の重複や不整合を防ぐため `deleteProtectedRange` → `addProtectedRange` の順で |

#### 実装方針

```
コピー完了（既存のcopyノード）
　↓
Google Drive Share（writerとして対象ユーザーに付与）
　↓
HTTP Request：spreadsheets.get でシート一覧と既存 protectedRanges を取得
　↓
Code ノード：batchUpdate 用 requests を生成
　　- 全シートの既存 protectedRanges を deleteProtectedRange
　　- 全シートを addProtectedRange（editors.users: auto-sales@wk-hd.com のみ）
　　- 編集可能シートのみ unprotectedRanges に指定セルを追加
　↓
HTTP Request：spreadsheets.batchUpdate 実行
```

**重要：** コピー先の `spreadsheetId` はコピーノードの出力（`$json.id`）を使う。固定URLは不可。

#### 修正ファイル

**`WF00_日時マスタコピー.json`** — 各コピーノードの後続に以下を追加

```
copy_T3＿営業数値フィードバック_ダッシュボード（個人）
　↓ Wait
　↓ [追加] HTTP Request: spreadsheets.get
　↓ [追加] Code: batchUpdate requests生成（個人設定）
　↓ [追加] HTTP Request: batchUpdate実行

copy_T3＿営業数値フィードバック_ダッシュボード1（店舗）
　↓ Wait1
　↓ [追加] 同上（店舗設定）

copy_T3＿営業数値フィードバック_ダッシュボード2（エリア）
　↓ Wait2
　↓ [追加] 同上（エリア設定）
```

事業部ダッシュボード（`WF00_研修参画フォームの受信 (1).json` の `copy_T3＿営業数値フィードバック_ダッシュボード`）も同様に追加。

#### Codeノードの実装

```javascript
// ダッシュボード種別ごとの編集可能セル設定
const editableConfig = {
  "ダッシュボード個人":  { "月次ダッシュボード": "E2:E3", "四半期ダッシュボード": "E2:E3", "半期ダッシュボード": "E2:E3", "年次ダッシュボード": "E2" },
  "ダッシュボード店舗":  { "月次ダッシュボード": "E2:E3", "四半期ダッシュボード": "E2:E3", "半期ダッシュボード": "E2:E3", "年次ダッシュボード": "E2" },
  "ダッシュボードエリア":{ "月次ダッシュボード": "E2:E3", "四半期ダッシュボード": "E2:E3", "半期ダッシュボード": "E2:E3", "年次ダッシュボード": "E2" },
  "ダッシュボード事業部":{ "月次ダッシュボード": "E2:E3", "四半期ダッシュボード": "E2:E3", "半期ダッシュボード": "E2:E3", "年次ダッシュボード": "E2" },
};

// A1記法 → API index変換
function a1ToIndex(a1) {
  const match = a1.match(/^([A-Z]+)(\d+)(?::([A-Z]+)(\d+))?$/);
  const colToIdx = c => c.split('').reduce((a, ch) => a * 26 + ch.charCodeAt(0) - 64, 0) - 1;
  const startCol = colToIdx(match[1]);
  const startRow = parseInt(match[2]) - 1;
  const endCol   = match[3] ? colToIdx(match[3]) + 1 : startCol + 1;
  const endRow   = match[4] ? parseInt(match[4])     : startRow + 1;
  return { startRowIndex: startRow, endRowIndex: endRow, startColumnIndex: startCol, endColumnIndex: endCol };
}

const dashboardType = "ダッシュボード個人"; // ← 各コピーノードに応じて変更
const config = editableConfig[dashboardType];
const sheets  = $input.first().json.sheets; // spreadsheets.get の返却値
const ADMIN   = "auto-sales@wk-hd.com";

const requests = [];

// 1. 既存 protectedRanges を削除
for (const sheet of sheets) {
  for (const pr of (sheet.protectedRanges ?? [])) {
    requests.push({ deleteProtectedRange: { protectedRangeId: pr.protectedRangeId } });
  }
}

// 2. 全シートを保護 + 編集可能シートのみ unprotectedRanges 付与
for (const sheet of sheets) {
  const sheetId   = sheet.properties.sheetId;
  const sheetName = sheet.properties.title;
  const editRange = config?.[sheetName];

  const protReq = {
    addProtectedRange: {
      protectedRange: {
        range: { sheetId },
        description: "auto-protect",
        warningOnly: false,
        editors: { users: [ADMIN] },
      }
    }
  };

  if (editRange) {
    protReq.addProtectedRange.protectedRange.unprotectedRanges = [
      { sheetId, ...a1ToIndex(editRange) }
    ];
  }

  requests.push(protReq);
}

return [{ json: { spreadsheetId: $('コピーノード名').item.json.id, requests } }];
```

## 論点・判断が必要な点

### 権限継承の実装方法

Google Driveのフォルダ権限は「上位フォルダに付与 → 下位へ自動継承」が機能する。
n8nのShareノードは1回の実行で1メールアドレスしか指定できないが、**アイテムが複数あれば同一ノードがアイテム数分繰り返し実行される**ため、複数人への付与は1ノードで対応可能。

```
例：エリア長3人のアドレスを持つ3アイテム
→ Share node（エリアマネージャー共有フォルダ）が3回実行 → 3人に付与
```

**承認済み仕様：** エリアマネージャー共有フォルダは全エリア長が共有するため、エリア長は他のエリアのデータも閲覧可能。同様に店長共有フォルダも全店長が共有するため、店長は他店舗のデータも閲覧可能。この仕様で確定。

### 個人ダッシュボードへの本人付与タイミング

現状の `WF00_日時マスタコピー` では、**既存フォルダには新規コピーしない（存在チェックあり）**。
→ Share は **コピー実施時のみ実行**される。再実行時にコピーが発生しないと権限も付与されない。
→ 初回コピー時に確実に実行されるため問題ないが、再コピーが必要な場合は別途対応が必要。

### リファレンスシートの更新タイミング

現行の `WF00_日時マスタコピー` はスケジュール実行。
→ リファレンスシートの更新をこのWFの末尾に追加すれば、毎回の実行で最新状態に保てる。

### エリア長向けリストの粒度

**確定：全エリア長が同一シートを参照（1シート全出し）。** エリア名カラムで各自がフィルタして使う運用。フォルダ権限と同様に、他エリアのリンクも技術的には閲覧可能な仕様。

### 【5】新メンバー追加時の権限付与

#### 課題

初回参画フォーム受信時（`WF00_研修参画`）にフォルダ作成・権限付与が行われるが、既存会社に**新メンバーが追加された場合**は再度フォームを受信しないため権限が付与されない。

#### 実装方針

`WF00_日時マスタコピー.json`（スケジュール実行）に、権限未付与ユーザーへの付与処理を追加する。

```
[スケジュールトリガー]
　↓
[マスタ全件取得]（既存）
　↓
[既存フォルダ検索]（既存 Search nodes）
　↓
[Share 各フォルダ] ← 追加（毎回全員に付与 or 未付与チェックあり）
```

#### ノード設定（フォルダ別）
